# 中小企業診断士試験 学習アプリ — 要件定義書

**版**: 1.0(2026-07-04) / **対象実装者**: Claude Code / **発注者**: 久保裕太郎

---

## 1. プロジェクト概要

中小企業診断士試験(1次・2次)の過去問学習アプリを開発する。本パッケージ(`data/`)には
**2015〜2025年度の全過去問を構造化したマスターデータ**(1次: 84試験・2,345問、2次: 44事例・181問)
が同梱されており、これを唯一のコンテンツソースとして使用する。

- **Phase 1**: React + Vite の PWA(スマホブラウザ最適化、オフライン動作)
- **Phase 2**: Capacitor で iOS ネイティブ化し App Store 配布(**個人利用が第一目的**)

## 2. 制約・前提(必読)

### 2.1 著作権
- 過去問の著作権は一般社団法人 中小企業診断協会(J-SMECA)に帰属する。同協会の利用規約
  (https://www.jf-cmca.jp/contents/016_riyoukiyaku.html)は私的使用の範囲での利用を許容するが、
  **不特定多数への公衆送信(App Store での一般公開等)は事前の許諾確認が必要**。
- したがって Phase 2 の公開範囲は「自分専用(TestFlight / 開発者配布)」を既定とし、
  一般公開する場合は協会への許諾照会を完了してから行う(先行事例: 過去問learning系サイト各社は許諾を得て運営)。
- アプリ内に出典表記を常設する: 「出典: 一般社団法人 中小企業診断協会 中小企業診断士試験問題」。

### 2.2 試験制度(令和8年度 = 2026年度)
- 1次試験: 2026年8月1日(土)・2日(日) / 2次筆記: 2026年10月25日(日) / **口述試験は廃止**
- 1次7科目: A経済学・経済政策(60分) B財務・会計(60分) C企業経営理論(90分) D運営管理(90分)
  E経営法務(60分) F経営情報システム(60分) G中小企業経営・政策(90分)。各100点、
  合格基準は総点数60%以上かつ各科目40%以上。
- 出典: 中小企業庁 令和8年度試験について
  (https://www.chusho.meti.go.jp/shindanshi/2026/260406shindanshi.html)、
  J-SMECA 試験日程(https://www.jf-cmca.jp/contents/010_c_/010_c_r08_shiken/R08_nittei.html)

#### 2.2.1 追記: 令和9年度(2027年度)試験カウントダウンについて
令和8年度試験終了後、アプリのカウントダウン対象を次回受験(令和9年度)に切替えた。
本稿執筆時点で中小企業庁・J-SMECAともに令和9年度日程は未発表(公式発表は例年3月上旬、
官報告示は4月上旬)。`src/lib/jst.ts` の `EXAM_DATE_1JI`/`EXAM_DATE_2JI` は過去の実施
パターン(1次=8月最初の土日、2次筆記=10月第4日曜)からの**推定値**(1次: 2027年7月31日
(土)・8月1日(日)、2次筆記: 2027年10月24日(日))であり、確定日程ではない。UI側は
`EXAM_DATE_CONFIRMED = false` を見て「予定・未確定」を明示する。公式発表後は日付と
このフラグ、および本節を更新すること。

## 3. データ資産仕様(`data/`)

| ファイル | 内容 | 件数 |
|---|---|---|
| `master_1ji.json` | 1次全問題(問題文・選択肢・正解・配点・タグ・図表フラグ) | 2,345問 |
| `exams_1ji.json` | 試験メタ(年度×科目、問数、配点合計、全員正解等の注記) | 84試験 |
| `taxonomy_1ji.json` | 論点タクソノミ(7科目57論点、keyword_v1) | 57論点 |
| `master_2ji.json` | 2次全事例(与件文・設問・配点・字数制約) | 44事例181問 |
| `sources_shushi.json` | 2次「出題の趣旨」公式ページURL(全11年度HTTP検証済) | 11件 |
| `manifest.csv` | 全原本PDFのURL・ローカルパス対応表 | 215行 |

### 3.1 1次問題スキーマ(master_1ji.json)
```json
{
  "id": "1ji-2024-B-Q05",
  "year": "2024", "subject": "B", "q": 5,
  "stem": "問題文…",                    // 設問分割がない場合
  "lead": "共通リード文…",              // (設問1)(設問2)形式の場合
  "items": [                             // 設問分割時のみ
    {"sub": 1, "stem": "…", "choices": {"ア": "…", "イ": "…"},
     "answer": "ウ", "points": 4, "all_correct": false, "no_scoring": false}
  ],
  "choices": {"ア": "…"},               // 非分割時
  "answer": "エ", "points": 4,
  "tags": ["財務・会計/企業価値"],
  "figure": {"flag": true, "pages": [7, 8]},   // 図表あり→頁画像参照
  "parse_status": "needs_review"        // 存在する場合のみ(§7参照)
}
```

### 3.2 重要な業務ルール
- **全員正解問題**(`all_correct: true`)は選択に関わらず正答扱い。**採点対象外**(`no_scoring: true`)
  は得点計算から除外(2021年G第24問など)。模試モードの採点ロジックに必ず反映すること。
- 2023年度は通常(`2023`)と再試験(`2023S`)の2回分が存在する。
- 図表問題(`figure.flag`)は468問(21%)。テキストだけでは成立しないため、
  **問題表示時に該当頁画像を併載**する(§5.3)。

## 4. 機能要件

### Phase 1(MVP、この順に実装)
1. **演習モード**: 科目→年度 or 論点タグで問題を選び、1問ずつ解答→即時正誤+正解表示。
2. **SM-2間隔反復**: 解答履歴(自己評価: もう一度/難/普/易)から次回出題日を算出。
   期日到来問題を「今日の復習」として提示。アルゴリズムは SuperMemo SM-2 準拠
   (EF初期値2.5、下限1.3)。
3. **模試モード**: 年度×科目を本番同一構成で出題。制限時間タイマー(科目別60/90分)、
   採点(§3.2ルール適用)、合否判定(60%/科目40%)。
4. **弱点ヒートマップ**: 論点タグ×正答率のマトリクス表示。タップでその論点の演習へ。
5. **学習統計**: 日別解答数、科目別正答率推移、試験日(2026-08-01)カウントダウン。

### Phase 1.5
6. **2次学習ビュー**: 与件文と設問の対面表示、字数カウンタ付き答案下書き(端末内保存)、
   公式「出題の趣旨」への外部リンク(sources_shushi.json)。
7. **検索**: 全文検索(問題文・選択肢・タグ)。

### Phase 2(iOS)
8. Capacitor ラップ、ローカル通知(復習リマインド)、ホーム画面ウィジェット(任意)。

## 5. 非機能要件

### 5.1 技術スタック(指定)
- React 18 + Vite + TypeScript / 状態: Zustand / 永続化: **IndexedDB(Dexie)**
- UI: Tailwind CSS。スマホ縦画面最適化(iPhone 13 mini 〜 Pro Max で崩れないこと)
- PWA: vite-plugin-pwa、全データ・図表画像をプリキャッシュし**完全オフライン動作**

### 5.2 性能
- master_1ji.json は約5MB。初回ロードで IndexedDB に取り込み、以後はネットワーク不要。
- 問題表示は体感即時(<100ms)。検索は Web Worker 化を検討。

### 5.3 図表画像
- `scripts/extract_figures.py` を実行すると figure_flag 問題の頁画像
  (`figures/{qid}_p{nn}.png`, 150dpi)が生成される(要: 原本PDF、manifest.csv参照)。
- アプリは図表問題で該当PNGを問題文の下に表示。画像は遅延ロード+キャッシュ。

## 6. データ品質の既知事項(実装時の扱い)

| 事項 | 件数 | アプリでの扱い |
|---|---|---|
| `parse_status: "needs_review"` | 1次52問 | 通常出題してよいが、問題表示に「原文確認」ボタン(頁画像)を必ず付ける |
| 本文中の `〓` | 微量 | 未解決グリフ(主に図中記号)。テキストは参考とし頁画像を正とする |
| 2015-2017(レガシー年度) | 21試験 | フォント欠陥をOCR文脈整合で全数解決済み。数値はOCR突合監査済み |
| 2次の図表フラグ | 未実装 | P2: 事例III/IVの添付図表検出は将来課題。当面 source_pdf 参照で代替 |

## 7. 開発の進め方

1. `data/` を読み込むローダーとスキーマ型定義(TypeScript)から着手
2. 演習モード(§4-1)を最短で動かし、実データで表示崩れ(長文・表・数式)を潰す
3. SM-2 → 模試 → ヒートマップの順。各マイルストーンでビルドが通る状態を維持
4. テスト: 採点ロジック(全員正解/採点対象外/合否判定)はユニットテスト必須
